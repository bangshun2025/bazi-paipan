> **版本**：v1.0  
> **日期**：2026-07-21  
> **作者**：产品经理  
> **状态**：待评审  
> **关联文件**：standalone.html（v0.8.1, 3220行, 单文件HTML零外部依赖）

## 一、概述

### 1.1 背景

v0.8.1 已具备基础的档案存取功能（localStorage，下拉选择器，回收站），但存在以下不足：

- **无小名字段**：自在班用户群体为儿童，日常以「小名」称呼，当前只有「姓名」；
- **无搜索能力**：档案增多后下拉选单效率低，无法按小名/姓名快速定位；
- **无独立档案视图**：必须将档案加载到表单才能看到排盘结果，无法「浏览」；
- **无预置数据**：每台设备需手动录入，自在班16孩数据未内置；
- **仅存表单数据**：保存的是输入参数，非排盘计算结果（八字/宫位），回显需重新计算。

v0.9.0 的目标是将「档案」从辅助工具升级为核心功能模块，成为用户管理八字档案的第一入口。

### 1.2 核心价值

| 维度 | 现状 | 目标 |
|------|------|------|
| 录入效率 | 逐人手动输入 | 预置16孩，开箱即用 |
| 查找效率 | 下拉滚动盲找 | 搜索 + 卡片列表一目了然 |
| 浏览体验 | 必须加载到表单重新排盘 | 点卡片直接看完整排盘 |
| 数据完整性 | 仅存参数，不存结果 | 参数+结果双存，秒开秒看 |

## 二、用户故事

### US1 — 小名录入与档案保存
> **作为** 自在班助教  
> **我想要** 录入孩子的小名和完整生日信息后保存为档案  
> **以便** 后续随时通过小名查找孩子的八字排盘

**验收条件**：
- 输入表单新增「小名」字段，与「姓名」并列
- 排盘后点击保存，小名+姓名+性别+生日+八字结果一并存入 localStorage
- 若小名为空，保存时提示「建议填写小名以便查找」，但不阻止保存

### US2 — 档案列表浏览
> **作为** 自在班助教  
> **我想要** 在一个美观的卡片列表中浏览所有已保存档案  
> **以便** 快速找到目标孩子而不需要逐个下拉

**验收条件**：
- 档案列表以卡片形式展示，每张卡片显示：小名、姓名、性别图标、八字四柱摘要、保存时间
- 卡片按最近保存时间倒序排列
- 列表支持滚动，适配移动端和桌面端
- 列表为空时显示引导文案「暂无档案，排盘后点击保存即可创建」

### US3 — 档案搜索
> **作为** 自在班助教  
> **我想要** 通过输入小名或姓名的关键词快速搜索档案  
> **以便** 在16+个孩子中秒级定位目标

**验收条件**：
- 搜索框位于档案列表顶部
- 输入即搜（无需按回车），模糊匹配小名和姓名
- 无匹配时显示「未找到匹配档案」
- 清空搜索框恢复完整列表

### US4 — 点击展开查看完整排盘
> **作为** 自在班助教  
> **我想要** 点击档案卡片直接查看完整八字排盘（含宫位标签）  
> **以便** 无需加载到表单、重新排盘就能看到结果

**验收条件**：
- 点击卡片展开/折叠完整排盘视图
- 展开后排盘以只读方式展示，样式与主排盘页面一致（含14种宫位标签）
- 展开的排盘数据来自档案中存储的计算结果，无需重新计算
- 同一时间仅一个卡片展开（手风琴模式），或可独立展开（由实现复杂度决定，优先手风琴）

### US5 — 预置自在班16孩数据
> **作为** 首次使用的助教  
> **我想要** 打开软件即看到自在班16位孩子的预置档案  
> **以便** 无需手动录入就能开始使用

**验收条件**：
- 首次加载（localStorage 无档案数据）时自动写入16孩预置档案
- 预置档案包含：小名、姓名、性别、生日、完整八字（四柱+胎元+命宫+身宫+首运+元辰）
- 预置档案与手动保存的档案无区别，可搜索、可展开
- 若用户已手动保存过档案，预置数据不覆盖已有数据
- 预置数据的保存时间记为首次导入的时间

### US6 — 从档案加载到表单
> **作为** 自在班助教  
> **我想要** 将某个档案的数据加载到排盘表单  
> **以便** 基于该档案修改参数后重新排盘

**验收条件**：
- 卡片上有「加载」按钮，点击后将小名、姓名、性别、生日回填到表单
- 加载后自动触发排盘
- 原有下拉选单的加载功能保留作为辅助入口

## 三、功能规格

### 3.1 数据模型

#### 3.1.1 档案对象（Archive）

```javascript
{
  id: Number,           // 时间戳，唯一标识
  nickname: String,     // 小名（新增字段，可为空字符串）
  name: String,         // 姓名
  gender: String,       // "男" | "女"
  // ---- 以下为表单输入参数（现有字段，保持不变）----
  year: Number,
  month: Number,
  day: Number,
  hour: Number,
  min: Number,
  prov: String,
  city: String,
  dist: String,
  useSolar: Boolean,
  calendarType: String,     // "solar" | "lunar"
  isLeap: Boolean,
  lunarMonth: Number|null,
  // ---- 以下为 v0.9.0 新增：排盘计算结果 ----
  bazi: {               // 八字四柱
    nian: { gan: String, zhi: String },   // 年柱
    yue:  { gan: String, zhi: String },   // 月柱
    ri:   { gan: String, zhi: String },   // 日柱
    shi:  { gan: String, zhi: String }    // 时柱
  },
  sanyuan: {            // 三垣
    tai:  { gan: String, zhi: String },   // 胎元
    ming: { gan: String, zhi: String },   // 命宫
    shen: { gan: String, zhi: String }    // 身宫
  },
  extras: {
    shengXiao: String,  // 生肖
    qiYun: Object,      // 起运信息（{year, month, day, hour, shun}）
    daYun: Array        // 大运列表 [{gan, zhi, startAge, endAge}]
  },
  gongWeiType: String|null,  // 当前选中的宫位类型
  // ---- 元信息 ----
  createdAt: String,    // ISO时间戳，创建时间
  updatedAt: String,    // ISO时间戳，最后更新时间
  isPreset: Boolean     // 是否为预置数据（新增，默认false）
}
```

#### 3.1.2 localStorage 键名

| 键名 | 用途 | 说明 |
|------|------|------|
| `bz_archives_v2` | 档案列表 | v0.9.0新键名，存储 Archive[] |
| `bz_trash_v2` | 回收站 | v0.9.0新键名，存储 Archive[] |
| `bz_archives` | 旧版档案（v0.8.x） | 首次加载时自动迁移到 v2 |

**迁移策略**：首次加载时检测 `bz_archives` 存在而 `bz_archives_v2` 不存在，则读取旧数据、补全新字段（nickname 默认取 name，bazi/sanyuan/extras 为空对象，isPreset=false），写入 `bz_archives_v2`，然后清除旧键。

### 3.2 功能模块

#### 3.2.1 表单增强

**新增字段**：

| 字段 | DOM ID | 类型 | 位置 | 说明 |
|------|--------|------|------|------|
| 小名 | `inNickname` | `<input type="text">` | 姓名左侧或右侧 | 选填，placeholder="小名" |

**行为变更**：
- `autoSaveArchive()` 和 `saveArchive()` 保存时包含 nickname 和完整 bazi 结果
- `saveArchive()` 保存时同步存储当前排盘计算结果（bazi, sanyuan, extras）

#### 3.2.2 档案面板（新增 UI 区块）

**触发方式**：在排盘结果区域下方新增「📋 档案」标签切换按钮，或新增一个可折叠的档案面板区域。

**布局**（由上到下）：

```

│ ┌──────────────────────┐   │

│ │ 庚子 丁亥 丁巳 戊申  │   │

└────────────────────────────┘
```

#### 3.2.3 卡片展开（排盘预览）

点击「展开」后，卡片下方内联渲染完整排盘表格（四柱+三垣），复用现有 `renderChart()` 的渲染逻辑，但 data 来自档案内存储的 bazi/sanyuan/extras 而非重新计算。

**状态管理**：手风琴模式——展开新卡片时自动折叠先前展开的卡片。

#### 3.2.4 搜索

- 搜索范围为 `nickname` + `name`，忽略大小写
- 使用 `String.includes()` 做模糊匹配，支持中文单字匹配
- 搜索词为空时显示全部
- 搜索无结果时显示空状态提示

#### 3.2.5 预置数据

**触发时机**：页面首次加载，且 `localStorage` 中 `bz_archives_v2` 不存在或为空数组。

**预置内容**：自在班16孩数据（详见第六节），每条档案的 `isPreset = true`。

**特殊处理**：
- 双胞胎（子旭/子阳、NONO/KK）八字数据完全相同，以独立档案分别存储
- 预置数据仅写入表单输入参数，bazi/sanyuan/extras 字段留空——用户首次点击「展开」时若检测到 bazi 为空则自动触发一次排盘计算并回填

## 四、与现有功能的兼容性

### 4.1 保留项

- 现有下拉选单（`archSel`）：保留，作为快速切换入口
- 现有保存/删除/回收站按钮：保留，功能不变
- 现有 `autoSaveArchive()` 机制：保留，排盘时自动保存
- 现有回收站功能：保留，数据迁移到新键名 `bz_trash_v2`

### 4.2 变更项

| 变更 | 旧行为 | 新行为 |
|------|--------|--------|
| 档案存储键名 | `bz_archives` | `bz_archives_v2` |
| 保存内容 | 仅表单参数 | 表单参数 + 排盘计算结果 |
| saveArchive() | 存表单数据 | 存表单数据 + 当前排盘结果 |
| 档案面板 | 无（仅有下拉） | 新增卡片式档案面板 |
| 输入表单 | 仅有姓名 | 新增小名字段 |

### 4.3 不兼容处理

- 旧 `bz_archives` 数据在首次加载时自动迁移，迁移后清除旧键
- 迁移后的旧档案 `bazi`/`sanyuan`/`extras` 为空对象，首次展开时自动计算并回填
- `isPreset` 字段旧档案统一标记为 `false`

## 五、非功能需求

### 5.1 性能

- localStorage 读写为同步操作，档案数量预期不超过100条，性能无瓶颈
- 搜索过滤使用 O(n) 遍历，100条以内无感知延迟
- 展开渲染复用现有 `renderChart()`，与主排盘渲染性能一致

### 5.2 存储

- 预估单条档案（含完整 bazi 结果）约 2-3KB
- 100条档案约 200-300KB，远低于 localStorage 5MB 限制

### 5.3 兼容性

- 与现有所有功能（双胞胎对比、AI录入、真太阳时、农历录入、宫位标签）无冲突
- 桌面端和移动端均需自适应

## 八、风险与依赖

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 旧版数据迁移丢失 | 低 | 迁移前做 localStorage 备份日志，迁移仅在首次执行 |
| 预置数据时辰不准确 | 中 | 文档中标注 hour=0 为占位值，提示用户自行调整；后续可补充精确时辰数据 |
| localStorage 容量不足 | 低 | 100条 ≈ 300KB，远低于 5MB 限制 |
| 与现有双胞胎卡片UI冲突 | 低 | 档案面板为独立区块，复用但不修改双胞胎卡片代码 |
| 移动端卡片列表过长 | 低 | 搜索过滤 + 手风琴模式减少同时可见内容 |
