## 需求演化记录

### v0.3.0: 双胞胎对比（实际实现）

**PRD 原始方案**（v0.2.0）: 双栏并排、大宝/小宝各一套大运流年
**实际实现**: 单表三栏（大宝 | 小宝 | 共享岁运）

**转向原因**: 同一对双胞胎八字相同，大运流年完全一致，没必要展示两次。横向对比八字差异 + 底部共享岁运表，一屏看完，更紧凑。

**PRD 偏差**: 布局方案变了，其余藏干替换规则不变。

## 当前版本: v0.3.0

两个方向，按优先级排列：

### P0 — 算法准确性

| 条目 | 说明 |
|------|------|
| 身宫边界 case | 中气/节气交界处身宫计算验证 |
| 真太阳时边界 | 子时跨日处理 |
| 自动化回归 | 参见 [`docs/TEST.md`](TEST.md) — 9 组已知八字 + 算法单元测试，一键验证 |

### P1 — 输出资产化

| 条目 | 说明 |
|------|------|
| 排盘截图导出 | 一键导出 PNG（含 chart + 岁运），用于报告/分享 |
| 排盘数据 JSON 导出 | 结构化数据，供 AI 分析或报告模板消费 |

### P2 — 功能增强

| 条目 | 说明 |
|------|------|
| 大运流年年份范围可调 | 当前固定 10 柱大运 + 10 年/柱流年，支持自定义 |
| 日柱空亡独立标注 | 当前空亡按年柱取，增加日柱空亡选项 |
| 合化提示 | 天干五合、地支六合/三合/三会标注 |

## 原则

1. **PRD 是意向，ROADMAP 是真相**——以实际实现为准，PRD 偏差在 ROADMAP 留痕
2. **发布优先**——每个版本必须有可交付的增量，不做纯重构
3. **算法不妥协**——排盘结果准确性是产品生命线，边界 case 优先于新功能
